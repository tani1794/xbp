ZOZOTOWNのサービスを
データの流れで理解する

ZOZOTOWNの画面では、商品を探す、買う、配送状況を見る、Q&Aを読む、セール通知を受け取る、 という機能が別々に見える。しかし裏側では、これらは全部つながっている。 たとえるなら、学校の出席簿、購買部の購入記録、掲示板のお知らせが一つのシステムでつながっている状態に近い。

購入系サービス

購入履歴・配送確認

ユーザーが商品を買うと、「誰が」「いつ」「何を」「いくらで」買ったかが保存される。 その後、注文データは配送データにつながり、「発送済み」「配送中」「到着予定」などを確認できる。

身近な例：
コンビニでレシートをもらうと、商品名・値段・日時が残る。 ZOZOTOWNでは、そのレシート情報がネット上のマイページに保存されるイメージ。
分析系サービス

検索履歴・おすすめ・ランキング

ユーザーが「黒 パーカー」「スニーカー」などを検索すると、その言葉が記録される。 その記録をもとに、似た商品や人気商品が表示される。 おすすめは偶然ではなく、過去の行動データから作られている。

身近な例：
図書館で何度もミステリー小説を借りる人に、司書さんが次のミステリー本をすすめるようなもの。
サポート系サービス

Q&A・セール通知・情報管理

よくある質問はFAQテーブルに保存される。 セール商品や再入荷情報は通知データとしてユーザーに送られる。 つまり、困った時の案内と、お得な情報の連絡もデータベースで管理されている。

身近な例：
学校の掲示板に「体育祭のお知らせ」や「忘れ物案内」が貼られるように、 ZOZOTOWNでは通知欄に情報が届く。
User 会員情報。すべての出発点。誰の行動なのかを判別する。
Search 検索履歴。ユーザーが何に興味を持ったかを記録する。
Product 商品情報。価格、在庫、ブランド、カテゴリを管理する。
Order 購入履歴。実際に買った商品を記録する。
Delivery / Notify 配送状況やセール通知につながる。

データベースのテーブルは、Excelのシートに近い。 たとえば「users」は会員名簿、「products」は商品リスト、「orders」は注文レシートの集まり。 フィールドは、そのシートの列名である。 つまり、テーブルは保管場所、フィールドは記録する項目と考えるとわかりやすい。

クエリでサービスが
実際に動く仕組みを見る

クエリとは、データベースへの質問文である。 たとえば「この人の購入履歴を見せて」「配送中の商品だけ出して」 「この人に合いそうな商品を探して」というお願いをSQLで書く。 つまりクエリは、大量のデータから必要な情報だけを取り出す注文書である。

購入履歴を表示するSQL 
-- 誰が、いつ、何を買ったかを表示する
SELECT
  users.name,
  orders.order_date,
  products.product_name,
  order_items.quantity,
  orders.payment_status
FROM orders
JOIN users
  ON orders.user_id = users.user_id
JOIN order_items
  ON orders.order_id = order_items.order_id
JOIN products
  ON order_items.product_id = products.product_id
WHERE users.user_id = 1
ORDER BY orders.order_date DESC;

高校生向けの説明

このSQLは、バラバラの表をつなげて一枚のレシートを作る処理。 usersは「誰が」、ordersは「いつ買ったか」、order_itemsは「何個買ったか」、 productsは「商品名」を持っている。 それぞれ別の表にある情報をJOINでつなぐことで、マイページの購入履歴が表示できる。

おすすめ商品を表示するSQL 
-- 検索履歴から興味がありそうな商品を出す
SELECT
  products.product_name,
  products.price,
  products.category_id
FROM products
WHERE products.category_id IN (
  SELECT category_id
  FROM search_logs
  WHERE user_id = 1
)
ORDER BY products.price ASC
LIMIT 10;

高校生向けの説明

これは、過去に検索したジャンルから似た商品を探す処理。 たとえば何度も「スニーカー」と検索している人には、 スニーカーカテゴリの商品を優先して出す。 友達が「この人はスニーカー好きだから、これも好きそう」と考えるのと同じ仕組み。

ランキングを作成するSQL 
-- よく売れている商品を順番に並べる
SELECT
  products.product_name,
  SUM(order_items.quantity) AS total_sales
FROM order_items
JOIN products
  ON order_items.product_id = products.product_id
GROUP BY products.product_id
ORDER BY total_sales DESC
LIMIT 10;

高校生向けの説明

このSQLは、売れた数を数えてランキングを作る処理。 文化祭の人気メニューランキングを作る時に、 「焼きそばが何個売れたか」「クレープが何個売れたか」を数えるのと同じ。 ZOZOTOWNでは商品ごとの販売数を集計して、人気順に並べる。
