SECTION 01

Flaskとは何か

Flaskは、Pythonを使ってWebアプリケーションを作るためのフレームワークである。 通常のHTMLは静的なページを作るのに向いているが、Flaskを使うとPythonで読み込んだデータや処理結果を Webページへ表示できる。

HTMLだけの場合

文字・画像・デザインの表示が中心。データ処理や条件分岐は苦手。

Flaskを使う場合

Pythonで処理した結果をHTMLに渡し、動的なWebページを作れる。

例1：最小のFlaskプログラム

app.py / minimum flask route
from flask import Flask

app = Flask(__name__)

@app.route("/")
def index():
    return "FlaskでWebページを表示しました"

if __name__ == "__main__":
    app.run(debug=True)

例3：CSVファイルを読み込んで表示する

app.py / csv to html
from flask import Flask, render_template
from csv import reader

app = Flask(__name__)

@app.route("/")
def index():
    with open("./mmcsv.csv", "r", encoding="utf-8") as csv_file:
        csv_reader = reader(csv_file)
        list_mm = list(csv_reader)

    return render_template("index.html", mmlist=list_mm)

if __name__ == "__main__":
    app.run(debug=True)
templates/index.html / jinja loop

例4：GoogleマップURLを生成する

app.py / google maps url
from flask import Flask, render_template
from csv import reader

app = Flask(__name__)

@app.route("/")
def index():
    with open("./mmcsv.csv", "r", encoding="utf-8") as csv_file:
        csv_reader = reader(csv_file)
        list_mm = list(csv_reader)

    for data in list_mm:
        if len(data) > 3:
            data[3] = "https://www.google.com/maps?q=" + data[3].replace("(", "").replace(")", "").replace(" ", "")

    return render_template("index.html", mmlist=list_mm)

if __name__ == "__main__":
    app.run(debug=True)
SECTION 04

Flaskで起きやすいエラー

Flaskでは、エラーの多くが Pythonの実行環境、 ファイル配置、 テンプレート名 に関係している。

エラー 主な原因 解決策
ModuleNotFoundError: No module named 'flask' Flaskが入っていないPythonで実行している AnacondaのPythonを選択して実行する
TemplateNotFound index.htmlがtemplatesフォルダにない templates/index.htmlに配置する
Port 5000 is in use 5000番ポートがすでに使われている control + Cで止める、またはport=5001を指定する
ページが真っ白 HTML側で変数を表示していない {{ 変数名 }} や for文で表示処理を書く
SECTION 05

今日の講義のまとめ

Flaskは、Pythonで処理した内容をWebページに表示するための橋である。 今回の講義では、URLにアクセスしたときに関数を実行する ルーティング、 HTMLを表示する render_template、 CSVを読み込んで画面に出す データ連携 を学んだ。

特に重要なのは、app.pyとtemplatesフォルダの役割分担である。 app.pyが処理を行い、index.htmlが表示を担当することで、PythonとWebページが接続される。
